# Project: Project Pack Automator (Referral Lead Engine)
# Tech Stack: React (Vite), Tailwind CSS, Lucide-React, Firebase (Auth, Firestore, Storage, Cloud Functions), Google Maps Static API, html2pdf.js / @react-pdf/renderer.

## Phase 1: Project Initialization & Infrastructure
1. Initialize a new React project using Vite.
2. Install and configure Tailwind CSS (global config, no inline styles).
3. Install Lucide-React for clean iconography.
4. Set up a Firebase project (Auth, Firestore, Storage).
5. Initialize Firebase Functions (Node.js) in a separate `/functions` directory.
6. Set up GitHub repo and GitHub Actions for CI/CD deployment to GitHub Pages.

## Phase 2: Database Design & Core Layout
1. Implement Firebase Authentication (protect all routes).
2. Build the app shell (Sidebar/Navbar) with a "Dashboard" and "Create Pack" view.
3. Define the Firestore `projects` document schema based on the PDF:
    - Metadata: Internal Ref (e.g., MKC-39000-62-1), Address, Planning Portal Ref.
    - Status: Approval Status, Approval Date.
    - Client Info: Homeowner Name, Contact Number (Manual entry).
    - Project Details: Description, Floor-by-floor breakdown.
    - Asset Links: Architectural Drawings Status, Structural Status, Finishes.
    - Media URLs: Cover Image (Zoopla), Aerial Map Image, Document Screenshots.

## Phase 3: The Scrapers (Cloud Functions)
1. Set up `axios`, `axios-cookiejar-support`, and `cheerio` in `/functions`.
2. Scraper 1 (Planning Portal): `scrapePlanningData`
    - Input: Council portal URL.
    - Extract: Address, Portal Ref No, Status, Date, Description, and an array of Document Names + URLs.
3. Scraper 2 (Zoopla/Property Image): `scrapePropertyImage`
    - Input: Zoopla or Rightmove URL.
    - Extract: The main high-res image URL of the property to be used for the Cover Page.

## Phase 4: Frontend Form & Data Aggregation
1. Build the "Create Project Pack" form with three main sections:
    - Auto-Fill Section: Inputs for Planning Portal URL and Zoopla URL. Add a "Fetch Data" button to trigger the scrapers.
    - Manual Override Section: Form fields to edit the scraped description, add the floor-by-floor breakdown, homeowner name, and phone number.
    - Status Toggles: Dropdowns for "Architectural Drawings" (e.g., Acquired), "Structural Calculations" (e.g., Speak to homeowner).
2. Implement Google Maps Static API integration: Automatically fetch a satellite aerial image based on the scraped address/postcode.
3. Save all aggregated data to the Firestore `projects` collection.

## Phase 5: Media & Document Management
1. Build a "Document Upload & Links" UI on the project page.
2. Allow the user to upload screenshots for specific document categories (Proposed, Existing, Decision Letter, Superseded).
3. Store these images in Firebase Storage and save their download URLs to the Firestore document.
4. Provide text inputs to attach the live URL links (pointing to the portal or Firebase Storage) for the buttons that say "[Link to Proposed Plans]" on the PDF.

## Phase 6: PDF Generation Engine (The Core Output)
1. Install `@react-pdf/renderer` (recommended over html2pdf for pixel-perfect, multi-page document control like your example).
2. Build the PDF template exactly matching the provided document:
    - Page 1: Zoopla Cover Image, Address, Internal Ref.
    - Page 2: Index/Contents.
    - Page 3-4: Summary, Description, Status, and Homeowner Contact block.
    - Page 5: Documents Index.
    - Page 6-10: Map through uploaded screenshots, inserting the image, title, and the clickable "[Link to...]" URL.
    - Page 11: Inject the Google Maps Static Aerial image.
    - Page 12: Technical Pack Boilerplate.
    - Page 13: Terms & Conditions Boilerplate (Commission terms, Non-Circumvention).
3. Add a "Generate & Download Pack" button to the frontend.

## Phase 7: Settings & Boilerplate Management
1. Create a "Settings" page in the app.
2. Allow the user to edit the global "Terms & Conditions" text and "Technical Pack" text here, saving it to a generic `settings` document in Firestore.
3. This ensures that if your commission structure or legal text changes, you update it once in the app, and all future PDFs reflect the new terms.
